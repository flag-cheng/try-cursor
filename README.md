# Cloud Agent 說明

本文件概述 Cloud Agent 的角色、運作流程與使用重點，協助你快速理解其運作模式。

## 核心概念
- Cloud Agent 在雲端執行，負責重計算、工具呼叫、外部資源存取。
- 本地端（例如 Cursor/IDE）專注於互動與檔案操作，並呈現 Cloud Agent 的結果。
- 兩端透過受控通道同步狀態；敏感或破壞性指令會被策略限制。

## 工作流程
1) 你在對話中提出需求，連同上下文送往 Cloud Agent。  
2) Cloud Agent 在雲端執行：可呼叫檔案讀寫、搜尋、linter、API 等工具。  
3) 執行結束後產出結構化結果（例如程式碼 diff、摘要、診斷）。  
4) 本地端套用結果並顯示；你可再發起下一輪指令形成迭代。

## 能力與分工
- 計算與生成：大型模型推論、批次任務、代碼生成。
- 工具整合：受控的檔案讀寫、搜尋、格式化、linter 等。
- 安全防護：拒絕長時間常駐或危險命令，避免未授權外部連線。
- 本地優勢：即時編輯、檔案瀏覽、與你的輸入互動。

## 安全與限制
- 禁止破壞性操作（如危險系統命令、未授權的網路連線）。
- 遵循時間限制（避免長時間阻塞）；長任務需分批執行。
- 受策略控管：只允許白名單工具與指令；輸出以結構化格式回傳。

## 使用建議
- 提供足夠上下文（目標、檔案路徑、限制）以提升結果品質。
- 對長流程分步下達指令，便於檢視每階段輸出。
- 如需特定格式（例如 PR 描述、測試計畫），在指令中明確要求。

## 常見問答
- **Cloud Agent 會直接改檔嗎？** 會以受控方式產生 diff 或檔案變更，由本地端套用。  
- **可以長時間跑服務嗎？** 不行，應拆分任務或改用適合的執行環境。  
- **若需外部 API？** 需在允許範圍內呼叫；未授權的外部連線會被阻擋。
